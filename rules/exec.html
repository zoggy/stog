<exec> - execute shell commands
Synopsis
<exec directory="" id="" in-xml-block="true" list="false"
      prompt="" show-code="true" show-stdout="true" stop-on-error="true"
> command </exec>
Description
Run the given command (or list of commands) and return the command output.
directory
If given, commands are executed in this directory. Else commands are executed in the current working directory.
id
If given and in-xml-block is true, the given id is given to the root XML element of the result.
in-xml-block [="true"]
Embed code and result in XML elements, except if set to false
list [="false"]
If set to true, a list of commands is expected in sub-elements, for example: 
<exec list="true">
  <x>echo foo > foo.txt</x>
  <x>ls</x>
  <x>ls -l</x>
</rule>
Commands are run in the given order. For each command, the result shows the command followed by its output.
prompt
The specified string is used as prompt string to print before the command in the output.
show-code [="true"]
Set to false not to have the original code in the result of the rule.
show-stdout [="true"]
Set to false not to have the output of the command in the result of the rule. Note that stderr is redirected to stdout during the execution of the command, so the output contains both.
stop-on-error [="true"]
Set to false not to stop compilation if an error occurs during execution of command.
